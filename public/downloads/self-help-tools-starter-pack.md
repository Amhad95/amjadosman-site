# Self-help tools starter pack

Use this short pack before running the free tools on amjadosman.vercel.app. It helps you bring better evidence into the input so the output can become a working decision document.

## 1. KPI audit input

- Current metric name and formula
- Current value and date range
- Owner and review meeting
- Decision the metric is meant to support
- What action follows when the number moves

## 2. Process map input

- Trigger and requester
- Every handoff and waiting point
- System used at each step
- Approval or exception rule
- Definition of a complete outcome

## 3. SOP input

- Process owner and backup
- Required inputs
- Ordered steps
- Quality check
- Escalation condition
- Template or record created at the end

## 4. Dashboard input

- Audience and decisions
- Reporting cadence
- Source systems
- Join keys
- Filters and drilldowns
- Alert thresholds

## 5. Implementation review

Before adopting an output, check:

1. Is every recommendation tied to a real owner?
2. Is the source data available and defined?
3. Does the workflow fit existing approvals?
4. What is the smallest pilot that can prove the change?
5. How will the team know the change worked?
